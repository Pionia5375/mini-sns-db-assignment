04비정규화 카운터의 trade-off

왜 like_count 컬럼을 따로 두는가.

❌ 정규화 only
SELECT count(*) FROM likes WHERE post_id = $1;

매 피드 조회마다 COUNT(*)
글 N개, 좋아요 M명이면 O(N × M)

✓ 비정규화 카운터
SELECT like_count FROM posts WHERE id = $1;

컬럼 한 번 읽으면 끝 O(1).
단, 카운터와 likes 행이 어긋날 위험.

해결책 → 좋아요 INSERT 와 UPDATE like_count 를 한 트랜잭션으로 묶는다.

성능은 비정규화가, 정합성은 트랜잭션이 떠받친다. 다음 세 슬라이드에서 코드로.
