06Transaction #2 · 팔로우 토글

한 액션이 세 갈래로 갈라진다.

// 데드락 방지: id 오름차순 잠금 const [first, second] = [me, other].sort(); await tx.query( "SELECT id FROM users WHERE id = $1 FOR UPDATE", [first]); await tx.query( "SELECT id FROM users WHERE id = $1 FOR UPDATE", [second]); // 1) follows 행 INSERT / DELETE await tx.query( "INSERT INTO follows (follower_id, followee_id) VALUES ($1, $2) ON CONFLICT DO NOTHING"); // 2) follower.following_count ±1 await tx.query( "UPDATE users SET following_count = following_count + 1 WHERE id = $1", [me]); // 3) followee.follower_count ±1 await tx.query( "UPDATE users SET follower_count = follower_count + 1 WHERE id = $1", [other]);
세 변화 한 단위

follows 행 + 두 사용자 카운터 ± 1. 모두 성공해야만 COMMIT, 하나라도 실패하면 ROLLBACK.

⚠ 데드락 시나리오

A→B 와 B→A 동시 팔로우.
순서 없이 잠그면 서로 기다리다 멈춤.

방지 패턴

두 사용자 행을 항상 id 오름차순으로 FOR UPDATE. → 순환 대기 자체가 성립 불가.
