05Transaction #1 · 좋아요 토글

INSERT 와 UPDATE 를 한 단위로 묶는다.

// app/actions/likes.ts — 트랜잭션 본문 await withTx(async (tx) => { const existing = await tx.query( "SELECT 1 FROM likes WHERE user_id=$1 AND post_id=$2", [uid, postId]); if (existing.rowCount) { // 이미 눌렀으면 해제 await tx.query("DELETE FROM likes WHERE …"); await tx.query( "UPDATE posts SET like_count = GREATEST(like_count - 1, 0) …"); } else { // 새 좋아요 — 동시 INSERT 는 ON CONFLICT 가 흡수 const ins = await tx.query( "INSERT INTO likes (user_id, post_id) VALUES ($1, $2) ON CONFLICT DO NOTHING", [uid, postId]); if (ins.rowCount) { await tx.query( "UPDATE posts SET like_count = like_count + 1 …"); } } });
PK 가 방어선

복합 PK (user_id, post_id) 가 동시 이중 INSERT 를 DB 단에서 거부.

멱등성

ON CONFLICT DO NOTHING + rowCount 검사로 이중 클릭을 흡수. 결과는 항상 동일.

원자성

likes 행과 카운터가 동시에 변하거나, 둘 다 안 변한다. 중간 상태가 존재하지 않는다.
