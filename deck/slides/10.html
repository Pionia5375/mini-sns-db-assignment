09통합 테스트로 입증한 정합성

npm run db:test — 세 시나리오, 모두 통과.

$ npm run db:test === 시나리오 1: 정상 후원 (alice → bob, 10P) === ✅ alice 잔액 -10: 195 ✅ bob 잔액 +10: 190 ✅ post tip_total +10: 10 ✅ ledger 행 +1: 4 === 시나리오 2: 잔액 부족 후원 (9999P) → ROLLBACK === ✅ 에러 발생: 잔액 부족: 195 < 9999 ✅ alice 잔액 그대로: 195 ✅ bob 잔액 그대로: 190 ✅ post tip_total 그대로: 10 ✅ ledger 행 그대로: 4 === 시나리오 3: 좋아요 토글 멱등성 === ✅ 첫 클릭: like_count +1: 3 ✅ 두 번째 클릭(해제): 원상복귀: 2 🎉 모든 트랜잭션 검증 통과
정상 후원

잔액 두 곳 + tip_total + ledger 모두 동시에 변한다. 중간 상태 없음.

잔액 부족 → ROLLBACK

에러 발생 후 네 상태 모두 그대로. 트랜잭션의 가장 직접적인 증거.

DB CHECK 제약

credit_balance >= 0, 자기 자신 팔로우, 중복 좋아요 — 모두 DB 단에서 거부.
